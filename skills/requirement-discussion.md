# Requirement Discussion

**Purpose:** Facilitate structured discussion to clarify project requirements, success criteria, and scope before starting data work.

**Inputs:** A user request or problem statement.

## Workflow

1. **Explore Context** (AUTO) — Understand the business context, existing data landscape, and stakeholder needs
2. **Propose Frame** (AUTO) — Propose a framing of the problem: what kind of analysis, what success looks like
3. **Gate: Review** (GATE) — Present the problem frame for human review and refinement
4. **Define Success** (AUTO) — Define measurable success criteria and acceptance thresholds
5. **Scope Analysis** (AUTO) — Map requirements to data needs, identify gaps, estimate effort
6. **Gate: Confirm** (GATE) — Present scoped requirements for final confirmation
7. **Archive** (AUTO) — Archive requirement artifacts for traceability

## Outputs

- Requirements document in describe/
- Success criteria checklist
- Scope and effort estimate
- Gap analysis report
