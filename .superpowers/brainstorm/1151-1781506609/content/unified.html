Unified Architecture: Two Layers, One System

Layer 1 tracks WHAT happened to the data. Layer 2 tracks WHY and WHAT WAS LEARNED.

The Complete Picture


┌──────────────────────────────────────────────────────────────────────────┐
│                        .datamind/                                         │
│                                                                           │
│  ═══════════════ LAYER 1: DATA LINEAGE (what happened) ═══════════════    │
│                                                                           │
│  graph.db              ← nodes: Dataset, Script, Execution                │
│                           edges: GENERATED_BY, PRODUCED, USED_INPUT       │
│                                                                           │
│  describe/*.md         ← auto data.describe for every dataset              │
│                           schema, stats, distributions, column meanings   │
│                                                                           │
│  scripts/              ← all AI-generated code, named by step             │
│                           the script IS the edge between datasets         │
│                                                                           │
│  ═══════════════ LAYER 2: COGNITIVE JOURNEY (why & what was learned) ═══   │
│                                                                           │
│  decisions.jsonl       ← {what, why, alternatives, implications, time}    │
│                           e.g. "forward fill, not interpolation, because  │
│                           stocks don't interpolate on non-trading days"   │
│                                                                           │
│  exploration.json      ← tree of approaches with status tags              │
│                           nodes: SELECTED | REJECTED | EXPLORATORY         │
│                           dead ends preserved — prevent repeating mistakes│
│                                                                           │
│  params.json           ← {script_id, run_id, key: value}                  │
│                           e.g. split_date: 2025-01-01                      │
│                           e.g. norm_source_A_min: -1.0                     │
│                                                                           │
│  discoveries.jsonl     ← {timestamp, tag, finding, linked_code, linked_data}│
│                           e.g. "Model acc drops 15% during earnings season"│
│                                                                           │
│  ═══════════════ LAYER 3: CONTEXT ASSEMBLY (bridge to AI) ═══════════════  │
│                                                                           │
│  context/              ← auto-generated from both layers                  │
│    PROJECT.md          ← project overview + active goals                  │
│    DATASETS.md         ← all datasets + lineage summary + schemas         │
│    HISTORY.md          ← recent decisions + discoveries + executions      │
│    EXPLORATION.md      ← active branch of exploration tree                │
│    PARAMS.md           ← current parameters for active models             │
│    CHECKPOINT.md       ← compressed state-of-project snapshot             │
│    CONTEXT_MANIFEST.md ← PRIORITY-ORDERED injection for new AI sessions   │
└──────────────────────────────────────────────────────────────────────────┘

How Layers Connect — A Real Example


     LAYER 1 (Lineage)              LAYER 2 (Cognition)
     ═════════════════              ═══════════════════

     raw/sales.csv                  decisions.jsonl:
        │                           → "Narrow 5 stocks → 2 (3 had missing data)"
        │ GENERATED_BY              → "Forward fill chosen over interpolation"
        ▼                           → "Split date 2025-01-01 (time-series)"
     scripts/clean_nulls.py
        │                           exploration.json:
        │ PRODUCED                  → Logistic Regression: REJECTED (52% acc)
        ▼                           → Random Forest: BASELINE (57% acc)
     processed/sales_clean.csv      → XGBoost: SELECTED (66% acc)
        │
        │ GENERATED_BY              discoveries.jsonl:
        ▼                           → "Sentiment scores on different scales"
     scripts/aggregate.py           → "Model fails during earnings (-15% acc)"
        │
        │ PRODUCED                  params.json:
        ▼                           → norm_source_A_min: -1.0
     processed/sales_agg.parquet    → split_date: 2025-01-01
                                    → xgb_max_depth: 6

     The AI sees BOTH layers in context/CONTEXT_MANIFEST.md.
     It knows the data flow AND the thinking behind every step.

Summary: The Five Innovations

Lineage
Graph + Script-as-Edge — Raw → Script → Processed. The script filename IS the relationship. Auto data.describe for every dataset.
Decisions
Decision Log — Every non-trivial choice: {what, why, alternatives, implications}. The rationale layer data lineage doesn't capture.
Exploration
Exploration Tree — SELECTED, REJECTED, EXPLORATORY nodes. Dead ends preserved to prevent repeating mistakes.
Parameters
Auto-Param Registry — Normalization bounds, split dates, hyperparameters. Extracted from scripts, keyed to execution runs.
Discoveries
Discovery Feed — Accidental insights, structured chronologically with tags, linked to data + code that produced them.
Assembly
Context Manifest — Priority-ordered, token-budget-aware injection assembled from all five sources. New AI sessions get curated understanding, not a dump.
