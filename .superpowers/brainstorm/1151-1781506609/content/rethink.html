Three Architecture Approaches

A

Full Web IDE

Custom web app with sidebar, chat, and code panels. Everything integrated into one interface. Like Jupyter + lineage tracking built from scratch.

Pros

  • Polished, controlled UX
  • Single entry point

Cons

  • Massive build effort
  • Locks users in
  • Can't use existing AI tools (Claude Code, etc.)
B

Context Engine + MCP Server

A backend-first system: context engine manages the knowledge graph, exposes it via MCP so ANY AI client can consume it. Web UI is a lightweight optional client.

Pros

  • Works with existing tools
  • Faster to build MVP
  • More flexible ecosystem

Cons

  • Less polished initial UX
  • MCP adoption still early
C

Hybrid: Engine First, UI Layered On

Start with the context engine + MCP server. Build the web UI incrementally as a first-class client. The engine is the product; the UI is one way to interact with it.

Pros

  • Best of both worlds
  • Validates core tech early
  • Progressive enhancement

Cons

  • Requires discipline to keep engine + UI decoupled
