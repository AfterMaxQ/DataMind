What the AI Actually Needs — Five Practical Innovations

1. The Decision Log — Rationale, Not Just Lineage

Data lineage tells you WHAT happened. Decision log tells you WHY. Every non-trivial choice is recorded: the options considered, the choice made, the rationale, and the downstream implications. Structure: {what, why, alternatives, implications, timestamp}. This is the single most important innovation — it's the missing layer between raw events and AI understanding.

2. The Exploration Tree — Dead Ends ARE the Context

Data exploration is a TREE, not a DAG. For every successful path, there were 3 dead ends. Dead ends are the most valuable context for future AI sessions because they prevent repeating mistakes. Each node has status: SELECTED, REJECTED, or EXPLORATORY. The AI sees the full tree, not just the winning path.

3. Auto-Parameter Registry — Capture What Code Alone Can't

Scripts contain logic but often hardcode parameters. A parameter registry auto-extracts: normalization bounds, split dates, hyperparameters, feature definitions. This is keyed to the script version AND the execution run. When the AI reuses a model, it has the exact parameters without re-reading the script.

4. The Discovery Feed — Unplanned Insights, Structured

Most valuable insights are ACCIDENTAL: "Model fails during earnings season." A chronological lab-notebook feed where every discovery — planned or accidental — is timestamped, tagged, and linked to the data/code that produced it. This is the analyst's thinking, structured for AI consumption.

5. The Context Manifest — Assembled, Not Dumped

When a new AI session starts, the system assembles a Context Manifest from #1-#4: project overview → dataset registry → active branch of exploration tree → recent decisions → relevant parameters → key discoveries. Priority-ordered, token-budget-aware. The AI gets a CURATED understanding, not a data dump.
